FRAMEWORK 01

ジョブ理論

顧客はプロダクトを「購入」するのではなく、進歩するために「雇用」する。

5つの問い

  1. その人が成し遂げようとしている進歩は何か？（機能的・社会的・感情的側面）
  2. 苦心している状況は何か？（誰が・いつ・どこで・何をしている時）
  3. 進歩を阻む障害物は何か？
  4. 不完全な解決策で我慢していないか？
  5. より良い解決策のためにトレードオフにしていいものは何か？

活用例: 学生が「英語学習で困っている人向けのアプリ」と言ったら

❌ そのまま進めさせる

✅ 以下の順番で問いかける：

  1. 「英語学習で困っている人って、具体的に誰？」（問い②）
  2. 「その人は今どうやって勉強してる？」（問い④）
  3. 「それでなぜ上手くいってないの？」（問い③）
  4. 「本当に求めている進歩って何？」（問い①）
  5. 「その進歩のためなら、何を犠牲にしてもいい？」（問い⑤）
FRAMEWORK 02

構造化思考

基本フロー

キークエスチョン → 論点分解 → 仮説構築 → 検証

キークエスチョン: 「誰が」「どのような状態になっている必要があるか」を疑問形に

論点分解の4つの方法

  1. 文章を区切る — キークエスチョン自体を構成要素に分割
  2. フレームワーク活用 — 3C、4P、SWOT等で網羅性を担保
  3. 前提条件で分解（そもそも分解） — 「そもそもその判断の前提は？」
  4. 通念的概念で分解 — 現在/未来、ソフト/ハード、静的/動的 等

論理性の4チェック

  • 問いを外していないか？
  • 漏れがないか？
  • 飛躍がないか？
  • ファクトが正しいか？

活用例: 学生が「就活がうまくいかない人を助けたい」と言ったら

☞ 「構造化しよう。"就活がうまくいかない"って、具体的に何がゴール？"誰が、どのような状態になっていれば成功なのか"を疑問形にしてみて」

☞ 「じゃあそのキークエスチョンを解くために、明らかにすべきことは？論点に分解して」

☞ 「この3つのうち、一番本質的なボトルネックはどれ？そこにAIで解決策を当てよう」

FRAMEWORK 03

コンセプトの型

Who × What × How

  • Who — 名前がつけられるレベルで具体的に
  • Insight — 本人が気づいていない、行動の裏にある本当の動機
  • Benefit — 機能的価値 + 感情的価値 + 社会的価値
  • RTB — なぜ自分たちにそれが実現できるのか

活用例: 学生が「面接練習をAIでやるアプリ」と言ったら

  1. Whoを具体化: 「誰が使う？初めての面接で緊張する人？何社も落ちて改善点がわからない人？」
  2. Insightを探る: 「本当に欲しいのは"面接練習の場"？"自分の何がダメなのか客観的に知りたい"ってこと？」
  3. Benefitを言語化: 「機能的には"AI面接官と練習できる"。感情的な価値は？社会的な価値は？」
  4. RTBを確認: 「AIだから実現できる理由は何？"24時間""何回でも""客観スコア"」
FRAMEWORK 04

新規事業の4つの発想軸

  1. ニーズから — ペインポイント・アンメットニーズから
  2. シーズから — 既存アセット（今回はAI）を活かして
  3. バリューチェーンから — 川上・川下の収益プール
  4. メガトレンドから — 環境変化・技術変化
FRAMEWORK 05

伝わるプレゼンの型

基本構成

  1. 前提を揃える（聞き手のレベル・スコープを定める）
  2. 結論・主張（一言で。＋聞き手に期待するアクション）
  3. 根拠・理由（3つに絞る。客観的事実で構築）
  4. 補足情報（本編から切り離す）
  5. 結論の再提示＋アクション

相手が知りたいこと（課題の深刻さ）から始めて、自分が伝えたいこと（プロダクトの価値）につなげる。余計な情報を削ぎ落として一本道のストーリーを作る。

学生のプレゼンが「機能説明の羅列」になっていたら：

❌ 「まず画面を見せて、この機能があって、次にこの機能があって…」

✅ 「プレゼンの順番を変えよう」

  1. 「皆さん、○○で困った経験ありませんか？」（前提を揃える）
  2. 「私たちは○○を作りました。一言で言うと"△△"です」（結論）
  3. 「なぜこれが必要か。理由は3つ」（根拠）
  4. 「実際に見てください」— 一番"おっ"となるシーンから見せる（デモ）
  5. 「このプロダクトが広がれば、○○な世界が実現します」（アクション）
PRACTICE

フェーズ別 問いかけマップ

各フェーズで「この問いを投げれば学生の思考が動く」という問いかけ集。

Phase 1: 課題発見

ジョブ理論 + 構造化思考

序盤 — テーマ選定
問いかけ狙い
「直近1週間で、自分自身が一番イラっとしたことは？」原体験からテーマを出させる
「それ、周りの人も同じように困ってる？」個人的な不満→市場性のある課題へ
「一番"切実度"が高いのはどれ？」ペインの深さで選ばせる
中盤 — ターゲット具体化
問いかけ狙い
「その人に名前つけられる？年齢、職業、どんな生活？」ペルソナの具体化
「その人は今、どうやって解決しようとしてる？」既存の代替手段を把握
「その解決策の何が不満？時間？お金？効果？」ペインの解像度を上げる
終盤 — 仮説構築
問いかけ狙い
「ここまでの話を一文にまとめて」課題仮説の言語化
「その仮説が正しいかどうか、どうやって検証する？」仮説思考を鍛える
「本当の"進歩"って何？根本的な欲求は？」機能的→感情的・社会的ジョブへ

Phase 2: ソリューション設計

新規事業 + コンセプトの型 + 構造化思考

問いかけ狙い
「AIを使わなくても同じことできない？AIだから初めて可能になることは？」AIの必然性を確認
「Who × What × Howで説明して」コンセプトの整理
「最初の30秒で"おっ"となる機能はどれ？それだけ作ろう」MVP定義
「課題を3つの論点に分解して」構造的に考えさせる
「同じ課題の既存サービスは？違いは？」競争優位性の確認

Phase 3: プロダクト開発

行動の加速が最重要

問いかけタイミング
「それ、AIに聞いたら30分で終わらない？」手作業で時間をかけている時
「今一番不安なところはどこ？そこから潰そう」何から手をつけるか迷っている時
「初めて使う人は最初に何をする？その体験を作ろう」機能を作り込みすぎている時
「完璧じゃなくていい。60点で一旦見せて」なかなか形にならない時
「ユーザー1人に見せたら最初に何て言う？」Day2の磨き込みフェーズ

Phase 4: プレゼン準備

伝わるプレゼンの型

問いかけ狙い
「ショウゴさんは経営者として何を知りたいと思う？」聞き手視点での構成
「聞いた後にどう動いてほしい？」アクションの明確化
「結論を一言で。言えないならまだ整理できてない」クリスタライズ
「根拠は3つに絞ろう。事実ベースで」根拠の客観性
「補足情報は全部後ろに回そう。本編は一本道」情報の取捨選択
「ショウゴさんは ①課題の深さ ②作ると考えるのバランス ③チーム貢献 を見ている」評価観点のヒント
