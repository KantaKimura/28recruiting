BEFORE YOU START

課題とは何か

解決策を考える前に、まず「課題」を正しく定義する。課題とは現状と理想状態のギャップであり、そのギャップに架け橋をかけるのが解決策（プロダクト）だ。

現 状
今の状態
不満・我慢・
問題がある
ギャップ ＝ 課題
架け橋をかけるのが…
解決策・プロダクト
理想状態
本当に求める進歩
ジョブ理論の
「本当のジョブ」
⚠️ 重要：現状と理想状態が特定されて初めて、課題が存在する。理想状態を表面的に捉えると、的外れな解決策になる。
FRAMEWORK 01

ジョブ理論

顧客はプロダクトを「購入」するのではなく、進歩するために「雇用」する。表面的なニーズに惑わされず、顧客が本当に達成したい進歩（ジョブ）を見つけることが重要。

4つの問い

  1. その人が成し遂げようとしている進歩は何か？（機能的・社会的・感情的など複数の側面）
  2. 苦心している状況は何か？（誰が・いつ・どこで・何をしている時）
  3. 進歩を阻む障害物は何か？
  4. 不完全な解決策で我慢していないか？
実例 — ミルクシェイク

表面的なニーズ（味・見た目）ではなく、本当のジョブ（用事）を見つけたことで売上拡大につながった事例。

ステップ内容
失敗の施策 アンケートをもとにフレーバー・トッピングを改善 → 成果なし
観察で判明 朝の通勤中、車内でミルクシェイクを飲んでいる人が多い
本当のジョブ 「運転中の退屈を、手を汚さず、長く解消したい」
競合との比較 バナナ → すぐ食べ終わる。ドーナツ → 手が汚れる。ミルクシェイク → 手が汚れず長持ち
打ち手 「より長持ちするよう、ボリュームを増量」→ 売上拡大
学べること：顧客が「買っているもの」と「雇用している理由（ジョブ）」は別物。
味を改善しても売れなかったのは、ジョブが「退屈しのぎ」だったから。競合はバナナやドーナツで、他のミルクシェイクではなかった。

「進歩」はさまざまな側面から捉える

機能的な側面だけでなく、その奥にある感情的・社会的な本音まで掘り下げると、本当に求められるプロダクト像が見えてくる。

側面問いオムツの例
機能的何ができるか？尿漏れを防ぐ
感情的どう感じたいか？「夜泣きされずに自分もよく眠れる」
社会的どう見られたいか？「良い親だと思われたい」
より深い進歩まで特定した例：
「尿漏れを防ぐ」だけ訴求しても売れない。「夜泣きしないから、あなたもよく眠れる」に変えたら売上が伸びた（米国）。
中国では「尿漏れしないから、よく寝て脳が育って賢い子になります」と社会的側面（子どもへの期待）まで捉えてヒットした。
本当の進歩を特定する — 日経新聞 × 新規事業の例

「表面的な理想状態」で止まると、解決策が的外れになる。本当に求めている進歩まで掘り下げることが重要。

ターゲット表面的な理想本当に求める進歩
就活生 楽に・手軽に・わかりやすく
情報収集したい		 志望企業に入社したい
法人営業 業界ニュースを
理解したい		 担当企業から信頼を得て、
受注したい
本当の進歩から導かれる解決策：
単に「簡単な言葉で届ける」「動画にする」では解決できない。
就活生なら志望業界・企業のニュースを就活の文脈で活かす視座まで提供し、営業なら担当企業・業界のニュースを自社提案と紐づけて活用方法まで示す、というサービス設計になる。

活用例: 学生が「英語学習で困っている人向けのアプリ」と言ったら

❌ そのまま進めさせる

✅ 以下の順番で問いかける：

  1. 「英語学習で困っている人って、具体的に誰？」（問い②）
  2. 「その人は今どうやって勉強してる？」（問い④）
  3. 「それでなぜ上手くいってないの？」（問い③）

FRAMEWORK 02

コンセプトの型

Who × What × How

  • Who — 名前がつけられるレベルで具体的に
  • Insight — 本人が気づいていない、行動の裏にある本当の動機
  • Benefit — 機能的価値 + 感情的価値 + 社会的価値
  • RTB — なぜ自分たちにそれが実現できるのか
INSIGHT の具体例

インサイトとは、「〜したい、でも〜」という人間の矛盾。本人も言語化できていない、行動の裏にある本当の動機。これを言い当てると、プロダクトやメッセージが一気に刺さるようになる。

① OISIX（半冷凍ミールキット）

「15〜20分だけ調理が必要な半冷凍食品」がヒット。一見、なぜ？と思う。完全にチンするだけの方が楽で、ガチ料理の方が本格的。それなのに「ちょっとだけ調理する」製品が売れた。

インサイト：「料理に手間をかける時間はない。でも、手抜き料理だとは思われたくない（完全な手抜きはしたくない）」という母親の矛盾。

② Instagram ストーリーズ

ストーリーズ追加前、フィードの投稿は「一生残る」からおしゃれに仕上げなきゃいけないプレッシャーで誰も投稿せず、アクティブユーザーが伸び悩んでいた。ストーリーズ追加後に一気に拡大。

インサイト：「友達に自分の日常を発信して承認されたい。でも、それがずっと残り続けるほど完璧に作り込むのはしんどい」という矛盾。→ 24時間で消えるストーリーズが解決。

活用例: 学生が「面接練習をAIでやるアプリ」と言ったら

  1. Whoを具体化: 「誰が使う？初めての面接で緊張する人？何社も落ちて改善点がわからない人？」
  2. Insightを探る: 「本当に欲しいのは"面接練習の場"？"自分の何がダメなのか客観的に知りたい"ってこと？」
  3. Benefitを言語化: 「機能的には"AI面接官と練習できる"。感情的な価値は？社会的な価値は？」
  4. RTBを確認: 「AIだから実現できる理由は何？"24時間""何回でも""客観スコア"」
FRAMEWORK 03

新規事業の4つの発想軸

アイデアは「どこから発想するか」で質が変わる。4つの軸で視野を広げ、まずロングリストを作ることが第一歩。

AXIS 01

ニーズから発想する

正確なターゲットカスタマーを意識して、カスタマージャーニー等からペインポイント・アンメットニーズを探し出し、そこへのソリューションとして事業機会を見出す。

問いかけ例：
・「誰が」「いつ」「どんな場面で」困っているか？
・今の解決策で我慢・妥協していることは何か？
・「本当はこうなりたい」のにまだ叶っていないこと（アンメットニーズ）は何か？
AXIS 02

シーズ（自社アセット）から発想する

企業が保有しているアセットを活かして新しい事業機会を見出す。ハードなものだけでなくソフトなものも幅広く考えることがポイント。

種類例発想の起点
ハード 設備・工場・技術・特許 この技術・設備を別の顧客・市場に使えないか？
ソフト ブランド・顧客基盤・ノウハウ・データ この顧客との関係やデータを別の事業に活かせないか？
AXIS 03

川上・川下領域での機会を確認する

既存事業のバリューチェーンを書き出し、川上・川下の収益プールや魅力度を確認していく。

川上
素材・技術・
仕入れ領域
→
既存事業
自社の現在の
バリューチェーン
→

流通・販売・
アフター領域
問いかけ例：
・バリューチェーンのどのステップに最も大きな収益プールがあるか？
・どのステップが最も非効率・不満が多いか？
・川上・川下のプレーヤーの役割を自社が担えないか？
AXIS 04

環境変化・メガトレンドから発想する

社会・技術・規制・市場の大きな変化を起点に、「この変化で生まれる新しいニーズは何か？」「今まで不可能だったことが可能になるとしたら？」を問う。

変化の種類例
技術変化AI・生成AI・ロボティクス・量子コンピュータ
社会変化少子高齢化・共働き世帯増加・価値観の多様化
規制変化働き方改革・DX推進・カーボンニュートラル
市場変化グローバル化・産業構造の変化・新興国台頭

ショートリスト化 — アイデアを絞り込む

4軸で出したロングリストを、以下2ステップで評価して深掘りするアイデアを絞り込む。

STEP 1 — 課題の妥当性チェック（大前提）

アイデアを進める前に、課題そのものが本当に存在するかを確認する。ここが曖昧なままだと、どんなに良いソリューションでも刺さらない。

  • 「どこの誰が」困っているのか？名前がつくレベルで具体的か？
  • 「本当に」困っているか？我慢・妥協している証拠はあるか？
  • その課題のために今お金や時間をすでに使っているか？
STEP 2 — 事業ポテンシャル評価（2軸）

以下の2軸で評価し、右上のゾーン（魅力度高×競争優位性高）を優先的にショートリスト入りさせる。

評価軸チェックポイント
① 事業の魅力度
十分な市場規模・収益プール		 ・十分な市場規模・収益プールがあるか？
・市場は成長しているか？
・単価・LTVが十分確保できるか？
② 自社の競争優位性
市場でシェアを取れるか		 ・市場で勝つためのKSF（成功要因）は何か？
・そのKSFで他社より優れているか？
・自社アセット（技術・顧客・データ等）が活きるか？
整理の目安： 2軸を縦横に置いた9象限マトリクスで候補を整理する。「魅力度は高いが自社優位性が低い」場合は、アライアンスや補完関係で優位性を作れないかも検討する。
FRAMEWORK 04

サービスを一文で定義する

コンセプトを正確に言語化することで、設計の軸が定まり、プレゼンの説得力も上がる。以下の型に当てはめて「誰に・何を・なぜ」を一文に凝縮する。

定義の型

〔状況〕という状況で、〔課題〕という課題を持つ 〔対象顧客〕向けの 〔製品/サービス名〕という〔製品/サービスジャンル〕です。 これには〔利点〕という利点があります。 〔既存の競合・代替品〕とは違い〔差別化要素〕が備わっています。

各要素の解説

要素問いかけ
〔状況〕対象顧客の1日の中で、その課題が生まれる具体的なシーンはどこか？（いつ・どこで・何をしている時）
〔課題〕その状況でユーザーが感じているペイン・フラストレーションは何か？
〔対象顧客〕名前がつけられるくらい具体的に、誰か？
〔製品/サービス名〕プロダクトの名前・呼び方
〔製品/サービスジャンル〕アプリ・プラットフォーム・コーチング・マーケットプレイス等
〔利点〕ユーザーにとっての機能的・感情的便益は何か？
〔既存の競合・代替品〕今ユーザーが代わりに使っているもの・方法は何か？
〔差別化要素〕競合・代替品と比べて際立って優れている点は何か？

メンターの使い方：

  1. 学生がアイデアを話し始めたら、この型に当てはめるよう促す
  2. どの要素が曖昧か特定し、そこに絞って問いかけを深める（ジョブ理論と組み合わせると効果的）
  3. 一文を完成させてからプレゼン構成・スライド設計に入るよう誘導する
FRAMEWORK 05

伝わるプレゼンの型

基本構成

  1. 前提を揃える（聞き手のレベル・スコープを定める）
  2. 結論・主張（一言で。＋聞き手に期待するアクション）
  3. 根拠・理由（3つに絞る。客観的事実で構築）
  4. 補足情報（本編から切り離す）
  5. 結論の再提示＋アクション

相手が知りたいこと（課題の深刻さ）から始めて、自分が伝えたいこと（プロダクトの価値）につなげる。余計な情報を削ぎ落として一本道のストーリーを作る。

学生のプレゼンが「機能説明の羅列」になっていたら：

❌ 「まず画面を見せて、この機能があって、次にこの機能があって…」

✅ 「プレゼンの順番を変えよう」

  1. 「皆さん、○○で困った経験ありませんか？」（前提を揃える）
  2. 「私たちは○○を作りました。一言で言うと"△△"です」（結論）
  3. 「なぜこれが必要か。理由は3つ」（根拠）
  4. 「実際に見てください」— 一番"おっ"となるシーンから見せる（デモ）
  5. 「このプロダクトが広がれば、○○な世界が実現します」（アクション）
