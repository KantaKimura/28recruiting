評価基準

Must 4項目 + Type 2項目 = 計6項目
各項目を3段階（◎ / △ / ✕）で評価

MUST 01

成果への執着

◎ 非常にいい状態

「これで終わり」にしない。最終プレゼンの直前まで質を追い込んでいる。チームの議論が「まあこれでいいか」という空気になったとき、「もう一段上げられないか」と問いかける。ワークの冒頭で「何をもってゴールとするか」を自ら定義しにいく。

△ NGではない状態

与えられたタスクには真面目に取り組んでいる。手を抜いている様子はない。ただし、自分から基準を上げにいく動きはない。時間が来たら「できた範囲で出す」という姿勢。

✕ NG

途中で明らかに諦める。「時間ないし、こんなもんでしょ」という発言や態度。アウトプットの質に対するこだわりが見えず、完成させること自体が目的になっている。

MUST 02

素直さ

◎ 非常にいい状態

メンターや社員からのFBを受けて、Day2で明確に行動が変わっている。単に言われた通りにするのではなく、FBの意図を自分なりに解釈し、自分のやり方にアレンジして取り入れている。チーム内でも、他メンバーの良い意見に「それいいね、そっちでいこう」と自然に乗れる。

△ NGではない状態

FBに対して「なるほど」と反応はするが、行動の変化が限定的。部分的には取り入れているが、根本的には自分のやり方を変えていない。悪気はないが、吸収のスピードが遅い。

✕ NG

FBに対して防御的になる。「いや、それは○○だから」と言い訳が先に出る。あるいは表面的に「わかりました」と言いながら、次のアウトプットが何も変わっていない。

MUST 03

自責思考

◎ 非常にいい状態

ワークがうまくいかなかったとき、真っ先に「自分が何を変えればいいか」の話をする。「さっきの議論が散らかったのは、自分が最初に論点を整理すべきだった」のように、改善の主語が自分。振り返りでチーム全体の結果に対して当事者として語れる。

△ NGではない状態

自分の担当部分についてはちゃんと責任を感じている。ただし、チーム全体の結果については「自分の範囲は頑張った」で止まる。当事者意識のスコープが自分の担当に閉じている。

✕ NG

うまくいかなかった原因を外に求める。「チームメンバーが」「時間が」「お題が」が主語になる。最悪のケースは、チーム内で他のメンバーのせいにする発言が出ること。

MUST 04

チームワーク

◎ 非常にいい状態

チームの成果を最大化するために自分の役割を柔軟に変えられる。発言が少ないメンバーに意見を求める。自分が得意でない領域は素直に任せる。必要であれば空気を壊してでも異論を言える。プレゼン後に他者の貢献に自然に言及する。

△ NGではない状態

チームの中で自分の役割は果たしている。協力的で揉め事は起こさない。ただし、チームの力を引き出す動きはなく、自分の持ち場を守っている印象。

✕ NG

独走型（合意を無視して一人で進める）、フリーライド型（議論に参加せず他人の成果に乗る）、偽の協調型（衝突を恐れ全てに同意）、政治型（自分が目立つためにチームを利用）のいずれかに該当。

TYPE 05

意思決定力（牽引型の特徴）

◎ 非常にいい状態

チームの議論が止まったとき、最初に「こっちでいこう」と方向を示す。情報が不十分でも仮説を立てて判断できる。決めた後にブレない。「まずやってみて、ダメなら変えよう」と前に進められる。

△ NGではない状態

自分で判断する場面はあるが、大きな意思決定では様子を見る。他の誰かが方向を示した後に賛同して動くタイプ。

✕ NG

判断を先送りする。「もう少し情報を集めてから」「みんなどう思う？」で時間を消費。決めた後にすぐ覆す。

※参謀型の学生はここが低い場合でも一定許容。Type⑥とセットで判断する。

TYPE 06

構造化思考力（参謀型の特徴）

◎ 非常にいい状態

ワーク開始直後に「この問題は○○と○○に分解できる」と問題の構造を示す。議論が散らかったとき交通整理ができる。最終プレゼンのロジック構造が明快。物事を抽象化して本質を掴む力がある。

△ NGではない状態

論理的に考えようとする姿勢はある。ただし構造化が浅い。「要するに何が本質か」まで到達していない。プレゼンのロジックに飛躍がある。

✕ NG

思いつきベースで発言する。議論に構造がなく、論点が散らかっていることに気づいていない。最終プレゼンが「言いたいことの羅列」。

※牽引型の学生はここが低い場合でも一定許容。Type⑤とセットで判断する。

総合判定

S-class Must 4項目のうち3つ以上が◎ + Type いずれかが◎ A-class Must 4項目のうち2つ以上が◎ B-class 全項目が△以上（✕がない） 不合格 Must のいずれかが✕

原則：Must項目に✕が一つでもあれば、Typeがどれだけ高くても不合格。

Must 4つは「プログリットで活躍するための最低条件」。ここを妥協すると、入社後に組織の足を引っ張る。

※本基準は2daysインターン時点の初回評価。面接フェーズで改めて調整する前提。
