FRAMEWORK 01

ジョブ理論

顧客はプロダクトを「購入」するのではなく、進歩するために「雇用」する。

5つの問い

  1. その人が成し遂げようとしている進歩は何か？（機能的・社会的・感情的側面）
  2. 苦心している状況は何か？（誰が・いつ・どこで・何をしている時）
  3. 進歩を阻む障害物は何か？
  4. 不完全な解決策で我慢していないか？
  5. より良い解決策のためにトレードオフにしていいものは何か？

活用例: 学生が「英語学習で困っている人向けのアプリ」と言ったら

❌ そのまま進めさせる

✅ 以下の順番で問いかける：

  1. 「英語学習で困っている人って、具体的に誰？」（問い②）
  2. 「その人は今どうやって勉強してる？」（問い④）
  3. 「それでなぜ上手くいってないの？」（問い③）
  4. 「本当に求めている進歩って何？」（問い①）
  5. 「その進歩のためなら、何を犠牲にしてもいい？」（問い⑤）
FRAMEWORK 02

コンセプトの型

Who × What × How

  • Who — 名前がつけられるレベルで具体的に
  • Insight — 本人が気づいていない、行動の裏にある本当の動機
  • Benefit — 機能的価値 + 感情的価値 + 社会的価値
  • RTB — なぜ自分たちにそれが実現できるのか

活用例: 学生が「面接練習をAIでやるアプリ」と言ったら

  1. Whoを具体化: 「誰が使う？初めての面接で緊張する人？何社も落ちて改善点がわからない人？」
  2. Insightを探る: 「本当に欲しいのは"面接練習の場"？"自分の何がダメなのか客観的に知りたい"ってこと？」
  3. Benefitを言語化: 「機能的には"AI面接官と練習できる"。感情的な価値は？社会的な価値は？」
  4. RTBを確認: 「AIだから実現できる理由は何？"24時間""何回でも""客観スコア"」
FRAMEWORK 03

新規事業の4つの発想軸

  1. ニーズから — ペインポイント・アンメットニーズから
  2. シーズから — 既存アセット（今回はAI）を活かして
  3. バリューチェーンから — 川上・川下の収益プール
  4. メガトレンドから — 環境変化・技術変化
FRAMEWORK 04

伝わるプレゼンの型

基本構成

  1. 前提を揃える（聞き手のレベル・スコープを定める）
  2. 結論・主張（一言で。＋聞き手に期待するアクション）
  3. 根拠・理由（3つに絞る。客観的事実で構築）
  4. 補足情報（本編から切り離す）
  5. 結論の再提示＋アクション

相手が知りたいこと（課題の深刻さ）から始めて、自分が伝えたいこと（プロダクトの価値）につなげる。余計な情報を削ぎ落として一本道のストーリーを作る。

学生のプレゼンが「機能説明の羅列」になっていたら：

❌ 「まず画面を見せて、この機能があって、次にこの機能があって…」

✅ 「プレゼンの順番を変えよう」

  1. 「皆さん、○○で困った経験ありませんか？」（前提を揃える）
  2. 「私たちは○○を作りました。一言で言うと"△△"です」（結論）
  3. 「なぜこれが必要か。理由は3つ」（根拠）
  4. 「実際に見てください」— 一番"おっ"となるシーンから見せる（デモ）
  5. 「このプロダクトが広がれば、○○な世界が実現します」（アクション）
FRAMEWORK 05

サービスを一文で定義する

コンセプトを正確に言語化することで、設計の軸が定まり、プレゼンの説得力も上がる。以下の型に当てはめて「誰に・何を・なぜ」を一文に凝縮する。

定義の型

〔状況〕という状況で、〔課題〕という課題を持つ 〔対象顧客〕向けの 〔製品/サービス名〕という〔製品/サービスジャンル〕です。 これには〔利点〕という利点があります。 〔既存の競合・代替品〕とは違い〔差別化要素〕が備わっています。

各要素の解説

要素問いかけ
〔状況〕どんな社会・市場・ユーザーの環境が背景にあるか？
〔課題〕その状況でユーザーが感じているペイン・フラストレーションは何か？
〔対象顧客〕名前がつけられるくらい具体的に、誰か？
〔製品/サービス名〕プロダクトの名前・呼び方
〔製品/サービスジャンル〕アプリ・プラットフォーム・コーチング・マーケットプレイス等
〔利点〕ユーザーにとっての機能的・感情的便益は何か？
〔既存の競合・代替品〕今ユーザーが代わりに使っているもの・方法は何か？
〔差別化要素〕競合・代替品と比べて際立って優れている点は何か？

記入例（シャドテン）：

英語を話せるようになりたいが独学では継続できない社会人が増えているという状況で、「練習しても上達している実感がない」という課題を持つ忙しいビジネスパーソン向けのシャドテンという英語コーチングサービスです。これには毎日プロが発音・リズムをフィードバックしてくれるという利点があります。独学アプリや従来の英会話スクールとは違いシャドーイング特化 × 毎日の個別添削が備わっています。

メンターの使い方：

  1. 学生がアイデアを話し始めたら、この型に当てはめるよう促す
  2. どの要素が曖昧か特定し、そこに絞って問いかけを深める（ジョブ理論と組み合わせると効果的）
  3. 一文を完成させてからプレゼン構成・スライド設計に入るよう誘導する
